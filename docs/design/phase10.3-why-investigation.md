# Phase 10.3: WHY investigation

Phase 10.3 explains observed HOW evidence using immutable, provider-bound,
offline evidence. It creates candidate rationale only; canonical architectural
knowledge remains governed and human-reviewed.

Initial providers are Git history, repository documentation, code comments,
tests, authored awareness records, local scars, and stored architect answers.
Remote PRs, issues, chat, telemetry, and models are extensions.

Each provider captures an immutable snapshot before querying it. Results retain
raw evidence, provider identity/version, query and target bindings, coverage
(`searched`, `partial`, `unavailable`, `not_configured`), and contradictions.
Unsearched and partial coverage never imply absence. Historical evidence never
becomes inferred rationale or a canonical claim without governed promotion.
